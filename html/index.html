Mission Control Language

If you can describe how to think,
you can build an AI that does.

MCL composes AI experts into structured reasoning workflows — multi-agent debate, iterative self-refinement, constitutional critique. The person who understands the problem can now build the system.

// Multi-Agent Debate — Du et al., 2023
// Three independent perspectives converge on a stronger answer.

let question = "Can LLMs truly reason, or are they pattern matchers?"

mission Debate(question) = {
    parallel {
        Optimist
        Sceptic
        Pragmatist
    }
    -> Synthesiser
}

output(Debate)

llm Pragmatist parallel ✓ done ~340 tokens
llm Synthesiser sequence ✓ done ~490 tokens

// Self-Refine — Madaan et al., NeurIPS 2023
// Generate, critique, revise. Loop exits when the judge passes.

let topic = "whether AI research should be open or proprietary"

mission SelfRefine(topic) loop(3) = {
    Drafter
    -> QualityJudge
}

output(SelfRefine)
llm Drafter attempt 1 ✓ done ~420 tokens
llm · judge QualityJudge attempt 1 ✗ fail ~180 tokens
llm Drafter attempt 2 {{feedback}} ✓ done ~390 tokens
llm · judge QualityJudge attempt 2 ✓ pass ~175 tokens
→ 2 of 3 attempts · early exit · loop closed
// Hybrid LLM + Classical ML — CoE, 2024
// LLM extracts features → ONNX scores → LLM interprets.

    -> ExtractFeatures

    -> LLMInterpreter

How MCL works
01 — THE FILE IS THE ARCHITECTURE
What you write is what runs

A mission.mcl file is not configuration for a system — it is the system. The pipeline structure you read is the pipeline structure that executes. No hidden orchestration layer. No framework translating intent into behaviour.

02 — EXPERTS ARE COMPOSABLE
Sequential, parallel, looping

Experts compose with -> for sequence, parallel {} for concurrent calls, and loop(N) for iterative refinement. A role: judge expert gates the loop — writing {{feedback}} on failure.

03 — NEUROSYMBOLIC BY DESIGN
Five kinds. One pipeline.

kind: llm for generation and reasoning. kind: rule for deterministic symbolic checks. kind: onnx for calibrated classical ML. kind: json_extract to bridge LLM output into typed context. Mix them in any order.

For everyone

Anyone can AI.

You don't need to be a developer to build structured AI reasoning. If you can describe how experts should think and talk to each other, you can build it in MCL — and run it against Claude, OpenAI, or any provider you choose.

Write down who the experts are and how they pass reasoning to each other. The runtime handles the rest — concurrency, context, provider calls.

1
Describe your experts — each one gets a plain-English expert.md file. No code.
2
Write the mission — declare who runs, in what order, in parallel or in sequence.
3
Run it — one command. Works with Claude, GPT-4o, or any OpenAI-compatible model.

For developers

One command turns any mission into an API.

forge serve exposes an OpenAI-compatible endpoint. Call it from Python, JavaScript, curl — or drop it into any tool that speaks the OpenAI API.

Your mission runs as a local model. Point any OpenAI-compatible client at it. No special SDK, no new framework to learn.

Start the server
$ forge serve missions/concepts/debate/agent.yaml

Expert kinds

Five expert kinds. One pipeline.

Each kind has a fixed contract. Combine them in any order. The runtime resolves each expert from its expert.md file.

kind: llm

LLM Expert

Any model behind an OpenAI-compatible API — Claude, GPT, Gemini, local Ollama. System prompt lives in expert.md.

Generation · reasoning · explanation
kind: rule

Rule Expert

Deterministic symbolic checks. Zero API calls. Microsecond execution. When a check fails, onFail is written to {{feedback}}.

Structural gates · compliance · format checks
kind: onnx

ONNX Expert

A trained classical ML model as a first-class pipeline expert. Local inference via ONNX Runtime — no API latency, no per-call cost.

Calibrated scoring · classification · local ML
kind: json_extract

JSON Extract Expert

Promotes keys from an LLM's JSON output into typed context variables. The bridge between neural generation and ML or rule inputs.

LLM → ONNX handoff · feature extraction
kind: http

HTTP Expert

Calls an external REST API and injects the response into pipeline context. Enrichment, lookup, or integration with existing services.

External APIs · data enrichment · webhooks
Concept missions

9 peer-reviewed reasoning patterns.
Each one a runnable file.

The academic literature on LLM reasoning describes architectures typically buried in Python orchestration code. These missions express each pattern in MCL — readable, runnable, and citable back to the paper that validated it.

Self-Refine
Generate, critique against a rubric, revise. Loop exits the moment the judge passes.
Madaan et al., NeurIPS 2023 · ~20% improvement over single-pass
loop(N) role: judge kind: llm
mission SelfRefine(topic) loop(3) = { Drafter -> QualityJudge }
Multi-Agent Debate
Independent perspectives generated in parallel, then converged by a synthesiser.
Du et al., 2023 · improved factuality & reasoning over single-agent
parallel {} kind: llm
mission Debate(question) = { parallel { Optimist Sceptic Pragmatist } -> Synthesiser }
Constitutional AI
Critique against explicit named principles stored in expert.md. The constitution is a markdown file.
Bai et al., Anthropic 2022 · more aligned outputs without human labels
kind: llm
mission ConstitutionalAI(task) = { Drafter -> ConstitutionalCritic -> Reviser }
Mixture of Agents
Sequential layered accumulation. Each expert receives the full output of all prior experts as context.
Wang et al., 2024 · medium-model MoA surpassed GPT-4 Omni on AlpacaEval 2.0
kind: llm
mission MixtureOfAgents(problem) = { Analyst -> Strategist -> Implementer -> Reviewer }
Hybrid LLM + Classical ML
LLM extracts semantic features, ONNX model produces calibrated scores, LLM explains. Each component does what it does best.
Composition of Experts, 2024 · heterogeneous expert routing
kind: llm kind: onnx kind: json_extract
mission HybridLLMML(content) = { LLMAnalyst -> ExtractFeatures -> Scorer -> LLMInterpreter }
LLM-as-Judge
Reference-guided quality evaluation. Rubric and reference answer live in the judge's expert.md — readable and forkable.
Zheng et al., 2023 · >80% agreement with human experts on MT-Bench
role: judge kind: llm
mission LLMAsJudge(question) = { Drafter -> ReferenceJudge }
Hallucination Reduction
Symbolic rule checks structural properties of LLM output. When a check fails, the exact violation feeds back as {{feedback}}.
RLSF, 2024 · symbolic certificates as correctness signals
kind: rule loop(N) kind: llm
mission HallucinationReduction(topic) loop(3) = { LLMDrafter -> FactChecker }
Verifiable Reasoning
LLM proposes a step-by-step reasoning chain. Symbolic rule verifies structural requirements. Fails feed back exactly what was missing.
AlphaGeometry, DeepMind 2024 · 42/50 IMO geometry problems
kind: rule loop(N) kind: llm
mission VerifiableReasoning(problem) loop(3) = { ProblemFramer -> ReasoningProposer -> StepVerifier -> ConclusionWriter }
Compositionality
Novel task decomposed into known primitives, run in parallel by specialised experts, assembled by a composer.
Dziri et al., NeurIPS 2024 · BAIR compound AI systems, 2024
parallel {} kind: llm
mission Compositionality(task) = { TaskDecomposer -> parallel { FactAnalyst NumberChecker ToneReviewer } -> Composer }
